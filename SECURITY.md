# Security policy

This repository begins as a private specification seed. Do not publish a suspected vulnerability, credential, exploit transcript, or sensitive trace in a public issue.

Use a private GitHub security advisory when available, or contact the repository owner through a private trusted channel. Include the affected commit, capability ID, permission/isolation profile, target environment, minimal reproduction, impact, and whether a credential or external side effect was involved. Replace real secrets with canaries.

Until the implementation passes `docs/security/threat-model.md` and `docs/quality/acceptance.md`, it must be treated as development software and must not be used on untrusted repositories or valuable hosts.

The seed repository grants no open-source license. A license may be selected explicitly before public release.
